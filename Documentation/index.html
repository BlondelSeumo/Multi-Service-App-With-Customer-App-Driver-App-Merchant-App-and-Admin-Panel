Firebase Configuration

1. Setup and create firebase project

You must create a New Project in Firebase by logging into Firebase Console and then entering using your Google account if you don't sign in, then create a new Project and specify the name you like for the project, then:

  • click on Add Firebase to your Android app
  • copy the application Id from your build.gradle and paste it as Android Package Name


  • click on Register App > Next
  • click on Download google-services.json. it will download a .json file we will use it later


you can skip the remaining setup steps

next up copy the downloaded google-services.json and paste it in the path 'Ornids\app\' to make sure that your app is linked with your Firebase Project

NOTE**: Register all applications, make sure you enter the customer, driver, and merchant app in the same firebase project

2. Setup firebase realtime database

Change database rules
  • click on Database > Realtime Database > Rules
  • change ".read": false and ".write": false to ".read": true and ".write": true
NOTE: if you did not change the firebase rules some of the app functions will not work like chat or save/change pictures.
  • where to find firebaseDB

3. Setup firebase storage

Change storage rules
  • click on Storage > Realtime Database > Rules
  • allow read, write;
NOTE: if you did not change the firebase rules some of the app functions will not work like chat or save/change pictures.

4. Setup login phone

next up you have to Enable Phone Authentication in Firebase Console by going to :

  • click on Authentication > Sign-In Methode > click on Phone
  • enable it and save

5. Get fcm server key

  • go to project setting
  • click on cloud messaging tab
  • now you get the fcm server key


6. How to insert sha1 fingerprint

in an Android project there are 2 fingerprints for each application, debug and release. Get the SHA1 Fingerprint then fill up SHA certificate fingerprints on firebase, you can follow the step below:

click here how to get android project SHA1

  • go to project setting
  • click on general tab
  • add new fingerprint


NOTE: if you upload your application to the playstore, you need to add playstore sha1 certificate here, if you don't add it the application doesn't work properly.

Android Configuration

In installing the Android application it will be more complex, follow the steps correctly so that the application can run properly, you can also watch the installation video in the Android application installation section for more details.

1. How to open project

  • open Android Studio > Open an Existing Android Studio Project > Select Your Project build.gradle file > ok


  • browse to directory where you place "Ornidsapp" project and click OK.

2. How to change package name

after you open the android project, please change the package name first

  • close / unselect “Compact Empty Middle Packages”


  • then, right click "refactor->rename" your package and rename it.
NOTE**: make sure the package name of each application matches what you entered when registering your application with firebase, click here how to register aplication on firebase

3. How to change app name

  • open Android Studio >app > res > values > strings.xml
  • enter your app name inside "app_name" string tag

4. How to change app icon

  • open Android Studio > app > res > all mipmap folder > ic_launcher.png
  • rename your icon with ic_launcher.png and paste in mipmap folder
  • replace all icons with particular size in particular folder

5. How to insert url

  • select project > go to Constans > constans
  • add your url http://yoururl.com in line 15


  • select project > go to res > xml > network_security_config.xml
  • add your url without http:// or https:// like "yoururl.com" in line 4

6. How to insert fcm server key

  • select project > go to Constans > constans
  • copy the fcm server key then paste it to constants, right on line 16.


  • click here how to find fcm server key

7. How to setting Mapbox

  • open Android Studio >app > res > values > strings.xml
  • enter your Mapbox access token "mapbox_access_token" string tag

  • open Android Studio > gradle scripts > gradle.properties
  • enter your Mapbox access token on mapbox "MAPBOX_DOWNLOAD_TOKEN"

8. Generate signed APK then create new key store

  • click build > generate signed bundle/APK




  • make new key store to get "JKS FILE"

9. Configure project structure

  • open project structure(ctrl+Alt+Shift+S) then create new signing config.


  • fill it with the key store and the "JKS FILE" that was created before.


  • click modules and than Click default config, choose your new signing config.


  • click build variants on the release option, choose your new signing config.


  • apply it and click "OK"
  • just ignore it if an error message appears on this step

10. How to get sha1 fingerprint

in an Android project there are 2 fingerprints for each application, debug and release.

  • run the signing report you will get a release and debug the SHA1 fingerprints




  • add your SHA1 fingerprints debug and release on each application and copy to each firebase project, click here where to insert the sha1 fingerprint on firebase
  • after entering all sha1 fingerprints for each application on firebase, download the google service json again and paste it on all applications
NOTE**: use the same google service json in all applications after you enter the sha1 fingerprint of all applications on firebase
  • clean and rebuild project.

4. SMTP configuration

Valid login details are required to send email using SMTP. You can see the explanation below.

  • SMTP Host = Name of the outgoing mail server.
  • Type of Encryption = None / SSL / TLS
  • SMTP Port = The port used for sending e-mail to the mail server
  • SMTP Authentication = Yes / No. In most cases this option must be enabled.
  • SMTP Username = Username of the login used for the mail server.
  • SMTP Password = Password used to login to the mail server.

Configure SMTP on your server

  • Please create a new email account via your cPanel hosting which will be used on later.
  • In cPanel hosting, please navigate to the Email Accounts section then click on Set Up Mail Client.


  • You can check SMTP details below. There are the details of the required SMTP, for the example in this tutorial using port 465, depending on the hosting service you are using.


  • remember the Outgoing Server and Port which will be needed later

Read Gmail messages on other email clients using POP

  • click here for more detail Gmail POP

Fill up the SMTP setting on admin panel

  • after you got all detail of your SMTP, fill up all the SMTP setting form on admin panel

Frequently asked questions

Why is my email being responded to late and not even getting a reply?

  • We will reply to incoming emails on “Monday” to “Friday” at 10:00 to 17:00 on Indonesian local time, we also take breaks on our national holidays. Please remember the schedule above when your email has not yet received a reply.
  • We will not reply to emails that contain spam, violence, sarcasm, and contain other negative content that can waste our time, constructive comments will be appreciated so we hope you can be wise in this case.
  • Please check your spam list or sometimes this can also caused by a failure of the email system either from our side or from the recipient’s email system, if this happens we will confirm you through the item comment section to send email with a different address.

Are there another author support contacts?

  • At this time we only provide support via email hello@ourdevelops.com and the codecanyon comment section.

Will it be charged again when there are updates of this item in the future?

  • Price changes will occur when there are additional features, price changes only apply to new buyers, you only need to pay the current price. There are no additional costs after purchasing the item, additional fees are only charged if you get installation and customization services from us.

If I want to get instalation services, what steps do I have to do?

  • Please send us an installation service request via email, please note that this service only covers basic installation only, if you want to change this application more advance according to your wishes please send a customization request.

If I want to get customization services, what steps do I have to do?

  • Please create and send us the list of customizations that you want in great detail so that the discussion process is more effective, we will determine whether we can do it or reject it. You can send this list before making a purchase to avoid disappointment when we reject your customization while you have purchased our item.

Why I didn’t get a verification code?

  • Make sure you have installed the application correctly, please follow the documentation that we have provided.
  • Make sure you have activated phone auth on firebase.
  • Make sure that your internet connection is stable.

Why is the application forced close when clicking on the topup button?

  • Please disable paypal in the admin panel when not in use, and fill in the paypal key form with “1234”.

Has created an account driver and customer but why can’t it find the driver?

  • Make sure you have activated the driver application, the button is on the application driver homepage.
  • Make sure the driver balance is sufficient according to the minimum balance requirements for each service on admin panel.
  • Make sure the driver is within the radius requirements for each service on admin panel.

Why my merchant does not appear on the customer application?

  • Make sure the merchant has at least 1 active menu.
  • Make sure the merchant balance is sufficient according to the minimum balance requirements for each service on admin panel.
  • Make sure the merchant is within the radius requirements for each service on admin panel.

Why my merchant does not appear on the customer application?

  • Make sure the merchant has at least 1 active menu.
  • Make sure the merchant balance is sufficient according to the minimum balance requirements for each service on admin panel.
  • Make sure the merchant is within the radius requirements for each service on admin panel.

Why i didn’t receive a driver / merchant registration confirmation email?

  • Make sure you have confirmed the account that you registered in the admin panel.
  • Make sure you enter your SMTP data correctly, you can use SMTP through the server or if you use gmail it requires special settings, check here

Why the list of merchant features and merchant categories does not load when registering?

  • When you do change the package name, automatically the url on the Android merchant changes according to your package name, this causes a connection failure between android and the server. Please replace the url again in utils > api > service > MerchantService which has changed “package name/...” to “merchant/...”.

Why merchant image didn’t appear in merchant applications?

  • When you do change the package name automatically the url on the Android merchant changes according to your package name, this causes a connection failure between Android and the server. please replace the url at constant > Constant which has changed “images/package name” to “images/merchant”.
